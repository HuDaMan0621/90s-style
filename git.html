Git != GitHub

Git

  • Git does not keep track of your files; it keeps track of changes.
  • Think of a Git repo as a tree. It is saved versions of your code.
    1. To add to the tree, you commit a snapshot of changes.
    2. A sequence of commits is called a branch.
    3. Files are committed from your working tree (your hard drive) to the master branch.
  • Cloning a repo gives you full functionality of the repo as if it were your own.

GitHub

  • GitHub is a backup for your commits.
  • GitHub is a useful tool for collaborating with other developers.
  • Forking a git repo gives you a personal copy of the repo that you can contribute to.
  • Merging and conflict resolution will be handled on GitHub.
  • Pull Requests: asking a collaborator to review AND merge your code.

Git Commands

  • git add adds working tree to index (staging area).
  • git commit adds index to branch. Use -am argument to add and commit simultaneously.
  • git push adds local branch to master.
  • git log shows a record of all commits.
  • git log --oneline is a more succinct verson of git log.
  • git init creates a repo in your current directory.
  • git clone copies a specified repo into your current directory.
  • git status shows current status of everything in your repo.
  • git branch shows you what branch you are currently working from. Use -d argument to delete branch.
  • git checkout allows you to switch branches.
  • git diff shows what was added to/deleted from the specific file.
